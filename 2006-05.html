_ [Eclipse][Java]コンパイラのバグ？

以下のコードをEclipse 3.1のコンパイラでコンパイルしてみると、 

public class Outer {
  private Bar instanceVar = new Bar();  
  public void createInnerAndCall() {
    new Object(){
      public void call() {
        /*
         * エンクロージングクラスのインスタンス変数の型を
         * 使ってstaticメソッドをcall．普通はBar.call()の
         * ようにして、呼び出すべき
         */ 
        instanceVar.call();
      }
    }.call();
  }  
  public static void main(String[] args) {
    new Outer().createInnerAndCall();
  }
}
class Bar {
  public static void call() {
    System.out.println("Bar");
  }
}

こんな感じのコードが生成される(Barクラスの生成コードは省略)のだが、これってJava仮想マシン仕様的にまずいコードな気がする。SunのjavacやJikesだと、エンクロージングクラスのprivateなメンバにアクセスするときは、access$0みたいな名前の(非privateな)アクセス用メソッドが生成されて、そのメソッド経由でprivateなメンバにアクセスするようなコードになるのだが、Eclipse 3.1のコンパイラが生成したこのコードでは、エンクロージングクラスのprivateフィールドに直接アクセスするようなコードになっている。なんかすごくバグっぽいのだが、報告しといた方がいいのかな。

_ [書籍][言語]Types And Programming Languages 〜P.25

3月に買ってP.6まで読んだ後、そのまま放置していたが、読むのを再開。とりあえず、P.25まで読んだ。ここまでの内容は、型システムの目的や言語の安全性の定義、コンピュータサイエンスにおける型システムの研究の大まかな歴史など。

ここまでの話はまだ概要といったところだが(当たり前)、意味がわからなかったり忘れている一般の(非技術用語の)英単語が結構あって、何度も辞書を引くことになった。これでは効率が悪い。英単語を勉強し直すべきなのかも。しかし、英単語を効率的に勉強するにはどうしたらいいんだろう。ぱっと思いつくのは、英単語帳を買って勉強することだが、モチベーションが続かなそうなのが問題だ。
